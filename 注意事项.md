# GitHub Trending Crawler - 注意事项

## ⚠️ 法律和合规性

### 1. GitHub 服务条款
- **风险**: 自动化爬虫可能违反 GitHub 的服务条款
- **建议**: 使用前仔细阅读 [GitHub 服务条款](https://docs.github.com/en/site-policy/github-terms/github-terms-of-service)
- **替代方案**: 优先使用 [GitHub API](https://docs.github.com/en/rest)

### 2. robots.txt 合规
- **要求**: 必须遵守 [github.com/robots.txt](https://github.com/robots.txt)
- **检查**: 确认允许爬取的路径
- **限制**: 遵守爬取延迟指令

### 3. 数据使用限制
- **版权**: GitHub 页面内容受版权保护
- **用途**: 仅限个人学习和研究使用
- **禁止**: 未经授权不得重新发布数据

## 🔧 技术风险

### 1. 反爬虫机制
- **IP封禁**: GitHub 可能封禁频繁请求的 IP
- **验证码**: 可能触发人机验证
- **解决方案**:
  - 使用代理 IP
  - 降低请求频率
  - 添加随机延迟

### 2. 页面结构变化
- **风险**: GitHub 可能随时更新页面 HTML 结构
- **影响**: CSS 选择器失效，爬虫无法解析数据
- **预防**:
  - 定期检查选择器有效性
  - 实现错误处理和日志记录
  - 准备备用解析方案

### 3. 网络稳定性
- **超时**: 网络请求可能超时
- **断连**: 连接可能中断
- **解决方案**:
  - 设置合理超时时间 (30秒)
  - 实现重试机制
  - 添加异常处理

## 🛡️ 最佳实践

### 1. 请求频率控制
```python
# ❌ 错误做法: 连续快速请求
for url in urls:
    requests.get(url)

# ✅ 正确做法: 添加延迟
import time
import random

for url in urls:
    requests.get(url)
    time.sleep(random.uniform(1, 3))  # 1-3秒随机延迟
```

### 2. User-Agent 设置
```python
# ❌ 错误做法: 使用默认 User-Agent
requests.get(url)

# ✅ 正确做法: 设置浏览器 User-Agent
headers = {
    "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"
}
requests.get(url, headers=headers)
```

### 3. 错误处理
```python
# ❌ 错误做法: 无错误处理
response = requests.get(url)
data = parse(response.text)

# ✅ 正确做法: 完整错误处理
try:
    response = requests.get(url, timeout=30)
    response.raise_for_status()
    data = parse(response.text)
except requests.RequestException as e:
    print(f"请求失败: {e}")
    return []
except Exception as e:
    print(f"解析失败: {e}")
    return []
```

## 🔄 替代方案

### 1. GitHub API (推荐)
```python
import requests

headers = {
    'Authorization': 'token YOUR_GITHUB_TOKEN',
    'Accept': 'application/vnd.github.v3+json'
}

# 搜索热门仓库
response = requests.get(
    'https://api.github.com/search/repositories',
    headers=headers,
    params={
        'q': 'stars:>1000',
        'sort': 'stars',
        'order': 'desc'
    }
)
```

**优点**:
- 官方支持，稳定可靠
- 结构化 JSON 响应
- 更好的速率限制
- 无需解析 HTML

## 🐛 常见问题

### Q1: 收到 403 Forbidden 错误
**原因**: GitHub 阻止了自动化请求
**解决**:
1. 检查 User-Agent 设置
2. 降低请求频率
3. 使用代理 IP
4. 切换到 GitHub API

### Q2: 返回空结果
**原因**: GitHub 页面结构变化
**解决**:
1. 更新 CSS 选择器
2. 检查 HTML 结构
3. 启用调试日志

### Q3: 连接超时
**原因**: 网络问题或 GitHub 限制
**解决**:
1. 检查网络连接
2. 增加超时时间
3. 实现重试机制

### Q4: 数据不准确
**原因**: 解析逻辑错误
**解决**:
1. 验证选择器正确性
2. 检查数据提取逻辑
3. 添加数据验证
