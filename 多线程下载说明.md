# GitHub Trending Crawler - 多线程下载功能

## ✨ 功能特性

### 🚀 多线程并发下载
- **并发控制**: 可配置线程数 (默认 5，建议 3-10)
- **智能调度**: 自动分配下载任务
- **资源优化**: 合理控制并发避免被封

### 📊 实时进度跟踪
- **进度条**: 使用 tqdm 显示美观进度
- **速度统计**: 实时显示下载速度
- **完成统计**: 成功/失败数量统计

### 🔄 断点续传
- **自动跳过**: 已下载文件自动跳过
- **智能检测**: 文件完整性检查
- **继续下载**: 中断后可继续

### 🛡️ 错误处理
- **自动重试**: 失败自动重试 (默认 3 次)
- **指数退避**: 重试间隔指数增长
- **超时控制**: 可配置超时时间

## 🚀 快速开始


```bash

python cli.py crawl -o my_repos.json

# 第二步: 下载仓库
python cli.py download my_repos.json -w 5 -n 10
```

### 方法 3: Python 脚本

```python

from downloader import RepoDownloader

# 爬取热榜
crawler = GitHubTrendingCrawler()

# 多线程下载
downloader = RepoDownloader(max_workers=5)
results = downloader.download_with_progress_bar(repos, "./downloads")
```

### CLI 参数

## 🔧 技术实现

### 多线程架构

```python
from concurrent.futures import ThreadPoolExecutor, as_completed

with ThreadPoolExecutor(max_workers=5) as executor:
    # 提交所有下载任务
    future_to_repo = {
        executor.submit(download_single, repo): repo
        for repo in repositories
    }

    # 处理完成的任务
    for future in as_completed(future_to_repo):
        result = future.result()
        update_progress(result)
```

### 进度跟踪

```python
class DownloadProgress:
    def __init__(self):
        self.total = 0
        self.completed = 0
        self.failed = 0
        self.start_time = None
        self.downloaded_bytes = 0

    def update(self, success: bool, file_size: int = 0):
        if success:
            self.completed += 1
            self.downloaded_bytes += file_size
        else:
            self.failed += 1
```

### 错误处理

```python
def download_with_retry(url, output_path, max_retries=3):
    for attempt in range(max_retries):
        try:
            response = requests.get(url, timeout=60, stream=True)
            response.raise_for_status()
            # ... 下载逻辑 ...
            return True
        except requests.RequestException as e:
            if attempt < max_retries - 1:
                time.sleep(2 ** attempt)  # 指数退避
            else:
                return False
```

## ⚠️ 注意事项

### 1. 并发控制
- **建议**: 3-10 个线程
- **过高**: 可能被 GitHub 封禁
- **过低**: 下载速度慢

### 2. 网络环境
- **国内**: 建议使用代理
- **带宽**: 根据带宽调整线程数
- **稳定**: 不稳定网络降低线程数

### 3. 磁盘空间
- **检查**: 下载前检查磁盘空间
- **清理**: 定期清理下载文件
- **压缩**: ZIP 格式节省空间

### 4. GitHub 限制
- **频率**: 控制请求频率
- **封禁**: 频繁请求可能被封
- **替代**: 考虑使用 GitHub API

## 🐛 常见问题

### Q1: 下载速度慢怎么办？

**原因**: 线程数过低或网络问题

**解决**:
```bash
# 增加线程数
python cli.py download repos.json -w 10

# 使用代理
export HTTP_PROXY=http://proxy:port
export HTTPS_PROXY=http://proxy:port
```

### Q2: 下载失败怎么办？

**原因**: 网络超时或仓库不存在

**解决**:
```bash
# 增加重试次数 (代码中修改)
downloader = RepoDownloader(retry_count=5)

# 增加超时时间
downloader = RepoDownloader(timeout=120)
```

### Q3: 如何跳过已下载的文件？

**解决**: 已下载文件会自动跳过

```python
# 检查文件是否存在
if os.path.exists(output_path):
    file_size = os.path.getsize(output_path)
    if file_size > 0:
        print("文件已存在，跳过")
        return True
```

### Q4: 如何限制下载大小？

**解决**: 使用 `-n` 参数限制数量

```bash
# 只下载前 5 个
python cli.py download repos.json -n 5
```
